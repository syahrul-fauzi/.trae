## Pedoman Kontribusi

- Gunakan semver untuk versi modul
- Semua PR harus menyertakan test dan dokumentasi
- Ikuti konvensi penamaan pada `NAMING_CONVENTION.md`
- Jaga keamanan: jangan commit secrets; gunakan `dotenv`

### Alur PR
- Buka issue
- Buat branch `feature/<slug>` atau `fix/<slug>`
- Tambahkan test, update docs
- Ajukan PR dengan deskripsi jelas dan checklist
